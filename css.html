What is CSS?

CSS (Cascading Style Sheets) is the language used to control the visual presentation of HTML documents. While HTML defines the structure and content of a page, CSS tells the browser how that content should look — its colours, fonts, spacing, sizes, layout, and animations.

CSS is called "cascading" because styles can come from multiple sources (browser defaults, external stylesheets, inline styles) and are applied in a specific priority order — the cascade. Understanding this cascade is essential to writing predictable CSS.

CSS is linked to an HTML file using a <link> tag in the head, written inside a <style> tag, or applied directly as an inline style attribute. Using an external stylesheet is best practice as it separates concerns and allows one CSS file to style an entire website.

/* External stylesheet linked in HTML head */ <link rel="stylesheet" href="style.css" /> /* Basic CSS rule syntax */ selector { property: value; another-property: value; }

CSS Selectors

A selector is the part of a CSS rule that identifies which HTML elements the styles should apply to. Choosing the right selector is crucial — overly broad selectors can accidentally break unrelated elements, while overly specific selectors can be hard to override.

Element Selector

Targets every instance of a given HTML tag on the page. Useful for setting global defaults but can be too broad for specific components.

p { color: #333; font-size: 16px; line-height: 1.6; }

Class Selector

Targets all elements with a specific class attribute. Classes are the most commonly used selector because they are reusable across multiple elements and specific enough to avoid side effects. A class is added to HTML with class="name" and selected in CSS with a dot prefix.

.card { background: white; border-radius: 8px; padding: 1.5rem; box-shadow: 0 2px 8px rgba(0,0,0,0.1); }

ID Selector

Targets a single element with a unique id attribute. IDs have very high specificity — they override class and element styles — so they should be used sparingly. Best practice is to use IDs for JavaScript hooks, and classes for styling.

#main-header { background: #1e3a8a; color: white; padding: 1rem 2rem; }

Descendant & Child Selectors

Descendant selectors (space) target elements nested anywhere inside another. Child selectors (>) only target direct children. These help you style elements contextually without adding extra classes.

/* Any <p> anywhere inside .card */ .card p { color: #555; } /* Only direct <li> children of <ul> */ ul > li { list-style: square; }

Pseudo-classes

Pseudo-classes target elements in a specific state — such as when the user is hovering over them, when an input is focused, or when a link has been visited. They are written with a colon after the selector.

/* Style a button when hovered */ button:hover { background: #1d4ed8; transform: translateY(-2px); } /* Style an input when it has focus */ input:focus { border-color: #2563eb; box-shadow: 0 0 0 3px rgba(37,99,235,0.15); } /* First item in a list */ li:first-child { font-weight: bold; }

The CSS Box Model

The box model is one of the most fundamental concepts in CSS. Every HTML element is rendered as a rectangular box. That box has four distinct layers, each adding space around the content:

  • Content — the actual text, image, or other content inside the element. Its size is controlled by width and height.
  • Padding — transparent space between the content and the border. Padding is inside the element, so it takes on the background colour.
  • Border — a line that wraps around the padding and content. You can control its width, style (solid, dashed, dotted), and colour.
  • Margin — transparent space outside the border. Margin creates distance between this element and its neighbours. Margins from adjacent elements can collapse into one.
/* Setting all four layers */ .box { width: 250px; padding: 20px; border: 2px solid #2563eb; margin: 30px auto; /* auto centres horizontally */ } /* box-sizing: border-box makes width include padding and border */ /* This is best practice and usually applied globally */ *, *::before, *::after { box-sizing: border-box; }
MARGIN — pushes other elements away
BORDER — visible outline
PADDING — space inside the border
CONTENT — your text or image

Typography in CSS

Typography is one of the most impactful aspects of web design. CSS provides fine-grained control over how text is displayed — from font choice and size to weight, spacing, and alignment.

body { font-family: 'Inter', Arial, sans-serif; /* Fallback stack */ font-size: 16px; /* Base size */ line-height: 1.6; /* Space between lines */ color: #1a1a1a; /* Text colour */ } h1 { font-size: 2.5rem; /* Relative to root font size */ font-weight: 700; /* Bold */ letter-spacing: -0.02em; /* Tight tracking for headings */ } .small-print { font-size: 0.75rem; color: #6b7280; text-transform: uppercase; letter-spacing: 0.1em; }

Using rem units (relative to the root font size) rather than fixed pixels makes typography scale correctly when users change their browser's font size setting — improving accessibility.

Colours & Backgrounds

CSS supports multiple ways to define colours — each with different use cases:

/* Named colours */ color: red; background-color: lightblue; /* Hex codes — precise, widely used */ color: #2563eb; background-color: #f9f9f7; /* RGB — useful when you need transparency */ color: rgb(37, 99, 235); background-color: rgba(37, 99, 235, 0.1); /* 10% opacity */ /* HSL — intuitive for adjusting lightness/saturation */ color: hsl(220, 90%, 56%); /* CSS Variables — define once, use everywhere */ :root { --primary: #2563eb; --bg: #f9f9f7; } .btn { background: var(--primary); } /* Gradients */ background: linear-gradient(135deg, #2563eb, #7c3aed); background: radial-gradient(circle, #eff4ff, #dbeafe);

Flexbox Layout

Flexbox (Flexible Box Layout) is a one-dimensional layout model that makes it easy to align and distribute elements along a row or column. It was designed to solve common layout problems that were awkward with older techniques like floats.

.container { display: flex; /* Activates flexbox */ flex-direction: row; /* row (default) or column */ justify-content: space-between; /* Horizontal alignment */ align-items: center; /* Vertical alignment */ gap: 1rem; /* Space between items */ flex-wrap: wrap; /* Allow items to wrap to next line */ } /* Control individual items */ .item { flex: 1; } /* Each item grows equally */ .featured { flex: 2; } /* This item takes twice the space */

Key Flexbox properties to remember: justify-content controls spacing along the main axis; align-items controls alignment on the cross axis; flex-wrap allows items to wrap onto multiple lines on smaller screens.

CSS Grid Layout

CSS Grid is a two-dimensional layout system — it handles both rows and columns simultaneously, making it ideal for building full page layouts.

.page-layout { display: grid; grid-template-columns: 250px 1fr; /* Sidebar + main content */ grid-template-rows: auto 1fr auto; /* Header + content + footer */ gap: 1.5rem; min-height: 100vh; } /* Card grid that auto-fills available space */ .card-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(220px, 1fr)); gap: 1rem; }

Responsive Design & Media Queries

Responsive design means building pages that look good and function well on all screen sizes — from large desktop monitors to small mobile phones. CSS achieves this through media queries, which apply different styles depending on the viewport width.

/* Mobile-first approach — default styles are for small screens */ .container { padding: 1rem; } .card-grid { grid-template-columns: 1fr; } /* Single column */ /* Tablet and above */ @media (min-width: 640px) { .card-grid { grid-template-columns: 1fr 1fr; } } /* Desktop and above */ @media (min-width: 1024px) { .container { max-width: 1200px; margin: 0 auto; } .card-grid { grid-template-columns: repeat(3, 1fr); } }

Mobile-first CSS means writing base styles for small screens and using min-width media queries to progressively enhance the layout for larger screens. This approach generally results in better performance on mobile devices.

Transitions & Animations

CSS transitions smoothly animate a property from one value to another when it changes state (e.g. on hover). CSS animations allow more complex, multi-step animations using keyframes.

/* Transition — smooth change on hover */ .btn { background: #2563eb; transition: background 0.2s ease, transform 0.2s ease; } .btn:hover { background: #1d4ed8; transform: translateY(-2px); /* Lift up slightly */ } /* Keyframe animation */ @keyframes fadeIn { from { opacity: 0; transform: translateY(10px); } to { opacity: 1; transform: translateY(0); } } .card { animation: fadeIn 0.4s ease forwards; }
