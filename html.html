What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. Every webpage you visit is built on an HTML foundation. It was created by Tim Berners-Lee in 1991 and has evolved through several versions — the current standard is HTML5.

HTML uses elements made up of tags to describe the structure and meaning of content. Tags are written inside angle brackets. Most tags come in pairs — an opening tag and a closing tag — with content placed between them:

<tagname>Content goes here</tagname>

Some elements are self-closing and do not wrap content — for example <img />, <br />, and <input />.

Basic HTML Page Structure

Every HTML page follows the same fundamental nesting structure.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>My Page Title</title> <link rel="stylesheet" href="style.css" /> </head> <body> <h1>Hello World!</h1> <p>This is my first webpage.</p> </body> </html>

Every HTML file must begin with <!DOCTYPE html> — this tells the browser to use the modern HTML5 standard. The <html> element is the root of the entire page. Inside it, <head> holds metadata (invisible to users) and <body> contains all visible content.

Common HTML Tags — Quick Reference

Below is a reference of the most commonly used HTML tags, what they are for, and a brief example of each.

Text & Headings

TagPurposeExample
<h1> – <h6>Headings — h1 is the largest, h6 the smallest. Only one h1 per page for good SEO.<h1>Page Title</h1>
<p>Paragraph of text. The browser adds space above and below automatically.<p>Hello world.</p>
<strong>Bold text — also signals importance to screen readers and search engines.<strong>Important</strong>
<em>Italic text — also signals emphasis in meaning.<em>Stressed word</em>
<br />Line break — moves the next content to a new line without starting a new paragraph.Line one<br />Line two
<hr />Horizontal rule — draws a dividing line across the page.<hr />
<span>Inline container — used to style or target a small piece of text with CSS or JS.<span class="red">word</span>

Links & Media

TagPurposeExample
<a>Anchor — creates a clickable hyperlink. The href attribute sets the destination URL.<a href="page.html">Click</a>
<img />Embeds an image. The alt attribute is required for accessibility — it describes the image for screen readers.<img src="pic.jpg" alt="A cat" />
<video>Embeds a video player with optional controls.<video src="clip.mp4" controls></video>
<audio>Embeds an audio player.<audio src="song.mp3" controls></audio>

Lists

TagPurposeExample
<ul>Unordered list — displays items with bullet points.<ul><li>Item</li></ul>
<ol>Ordered list — displays items numbered in sequence.<ol><li>Step 1</li></ol>
<li>List item — used inside <ul> or <ol> to define each item.<li>My item</li>
<dl>Description list — pairs terms with their definitions.<dl><dt>Term</dt><dd>Meaning</dd></dl>

Structure & Layout

TagPurposeExample
<div>Block-level container — groups elements together for layout or styling. Has no visual meaning on its own.<div class="card">...</div>
<span>Inline container — wraps a small amount of text or inline content for styling.<span style="color:red">text</span>
<table>Creates a data table. Uses <tr> for rows, <th> for header cells, <td> for data cells.<table><tr><td>Data</td></tr></table>

Form Elements

TagPurposeExample
<form>Container for all form elements. The action attribute sets where data is sent; method sets GET or POST.<form action="/submit" method="post">
<input />Interactive input field. The type attribute controls what kind: text, email, password, checkbox, radio, submit, etc.<input type="email" />
<label>Describes an input field. Linking it with a for attribute matching the input's id improves accessibility.<label for="email">Email</label>
<textarea>Multi-line text input. The rows and cols attributes set its initial size.<textarea rows="4"></textarea>
<select>Dropdown menu. Contains <option> elements for each choice.<select><option>Choice</option></select>
<button>Clickable button. type="submit" submits the form; type="button" is used with JavaScript.<button type="submit">Send</button>
<fieldset>Groups related form controls together visually and semantically.<fieldset><legend>Details</legend></fieldset>

Semantic HTML

Semantic HTML means using tags that describe the meaning of content, not just its appearance. Before semantic HTML5 tags existed, developers used generic <div> tags for everything. Semantic tags make pages more accessible to screen readers, easier for search engines to index, and more readable for other developers.

TagPurpose
<header>Introductory content for a page or section — typically the site logo, title, and navigation.
<nav>A block of navigation links — the main menu of the site.
<main>The dominant content of the page. There should only be one <main> per page.
<section>A thematically grouped block of content, usually with its own heading.
<article>Self-contained content that could stand alone — a blog post, news article, or product card.
<aside>Content related to the main content but not central to it — a sidebar or callout box.
<footer>Footer of the page or section — copyright info, contact links, etc.
